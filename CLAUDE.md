# 提瓦特人生重开模拟器 — 项目架构与进度

> 目标：制作一款文字类型的提瓦特（原神世界观）人生重开模拟游戏。
> 当前状态：Phase 1 完成 — 种族选择 + 增强天赋 + 7国体系，data/engine/ui 三层分离，纯前端运行。

---

## 2. 游戏流程 (Phase 1 当前)

```
标题画面 → 种族选择 → 国家选择 → 天赋抽卡+属性分配 → 逐年推进人生 → 死亡 → 人生总结 → 重新开始
```

### 2.1 种族选择 (RaceScreen) — Phase 1 新增
- 4个可选种族：人类/龙族/坎瑞亚人/天使族 (丘丘人仅作为诅咒结局)
- 种族决定：寿命类型(short/normal/long/immortal)、属性上限、特殊机制、可选国家范围
- 龙族仅可降生蒙德(雪山/高塔)、璃月(归离/魔神战争)、纳塔(火龙时代)
- 坎瑞亚人可在各国家早期时代降生，但有隐藏诅咒进度条
- 天使族仅可降生稻妻(白夜国/雷电真)、枫丹(雷穆利亚/审判庭)、蒙德(雪山)

### 2.2 国家选择 (NationScreen)
- 7国可选：蒙德(风)、璃月(岩)、稻妻(雷)、须弥(草)、枫丹(水)、纳塔(火)、挪德卡莱(冰)
- 可选国家由种族限制，不可选国家灰显
- 各3个时代，随机抽取（种族限制时代范围时在允许范围内随机）

### 2.3 天赋抽取与属性分配 (SetupScreen) — Phase 1 增强
- 从17个增强天赋中加权抽取10个（权重: common 60%, rare 30%, epic 10%）
- **保底机制**：前9抽无史诗→第10抽强制史诗
- 玩家选择2个天赋，史诗特殊事件线只能选1条
- 天赋新增字段：cap_modifiers(属性上限修正)、event_weight_modifiers(事件池权重)、storyline_flags(出生flag)、allowed_races/conflict_races
- 属性分配：基础10点+天赋奖励，每项默认上限5→天赋/种族可提升(硬上限10)
- UI显示 x/5 上限指示，达到上限时+按钮禁用

### 2.4 逐年推进 (GameScreen)
- 从0岁开始，每年触发一个事件
- 事件优先级：编年史(Chronicle) > 重大(Major) > 随机池(Pool,加权) > 兜底(Fallback)
- 随机池使用 pickWeighted，受天赋event_weight_modifiers和种族event_pool_tags影响
- 重大事件：d100属性检定 = clamp(60 + stat×6 - difficulty×7, 8, 95)
- 左侧面板显示：4属性条、诅咒/信仰进度条(按种族)、天赋、近期历史

### 2.5 特殊种族机制 — Phase 1 新增

**坎瑞亚人 — 诅咒侵蚀**
- curse_progress 每年被动 +1~4
- 60%+：体质每年-1
- 100%：死亡，flag "诅咒吞噬"，未来可解锁丘丘人延续

**天使族 — 信仰动摇**
- faith_loss 每年被动 -1（自然恢复）
- 事件"背弃信仰"选择可大幅增加
- 75%+：精神每年-1
- 100%：死亡，高天联系断裂

**龙族 — 自动龙裔线**
- 选龙族自动激活 specialLine: "dragon"，无需龙血余烬天赋

### 2.6 死亡与总结 (SummaryScreen)
- 死亡条件：体质<0、精神<0、事件致死、诅咒/信仰满、超过最大寿命
- Anthropic API 生成人生总结（失败则本地生成）
- 展示最终属性、天赋、种族信息

### 3.2 五种族

| 种族 | 寿命类型 | 特殊机制 | 可选 | 限制 |
|------|---------|---------|------|------|
| 人类 | normal | 无 | ✓ | 全部7国 |
| 龙族 | long(×1.4) | 自动龙裔线 | ✓ | 蒙德/璃月/纳塔的部分早期时代 |
| 坎瑞亚人 | normal | 诅咒侵蚀 | ✓ | 各国的早期时代 |
| 天使族 | long(×1.4) | 信仰动摇 | ✓ | 稻妻/枫丹/蒙德的早期时代 |
| 丘丘人 | short(×0.7) | 无 | ✗ | 仅诅咒结局触发 |

### 3.3 年龄阶段 (6阶段)
infant(0-5) → childhood(6-12) → teen(13-17) → youth(18-30) → middle(31-55) → elder(56-120)

### 3.4 核心属性 (4维, 范围 -30~40)
智力(intellect) / 精神(spirit) / 体质(physique) / 元素力(elemental)

### 3.5 寿命机制
- 基础寿命 = era.lifespanBase(53~61) + random(0,10)
- 乘以种族寿命倍率(short 0.7 / normal 1.0 / long 1.4)
- 加种族修正值 + 天赋寿命修正
- 最大寿命 = min(160, 最终寿命+30)
- 超过寿命阈值：每6年体质额外-1

## 4. 事件系统

### 4.1 事件类型
- **normal**：单线叙事，"下一年"推进
- **major**：2分支选择，d100属性检定，成功/失败不同结果

### 4.2 事件优先级
1. 编年史事件 (YEARLY_CHRONICLES: nation+era+age+flags精确匹配)
2. 重大事件 (SPECIAL_LINES 或 MAJOR_EVENTS年龄匹配)
3. 随机池 (YEARLY_POOLS + SPECIAL_LINES池，加权随机)
4. 兜底事件 (通用文本)

### 4.3 检定系统
`target = clamp(60 + stat×6 - difficulty×7, 8, 95)` 对 `roll(1,100)`

### 4.4 事件数据覆盖现状
- eventData.js: 5国×3时代×5年龄阶段的 normal + key 事件模板（纳塔/挪德卡莱待补充）
- lifeData.js: 完整蒙德高塔孤王0-81岁编年史 + 5国约4个重大事件 + 龙/天使特殊线

## 5. 技术架构

- **无构建工具**：IIFE + window全局变量 + `<script>` 标签加载
- **React 18** + Babel Standalone 浏览器端编译 JSX
- **Anthropic API** (claude-sonnet-4-20250514)：仅人生总结，非事件生成
- **暗黑幻想风**：深蓝黑底色 + 金色文字 + 星空粒子 + 打字机效果
- **Lore素材**：`reference/日月全事_月之六正式版_诗漱.md` (~500页原神世界观手册)
